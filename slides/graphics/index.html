Why do you want to do this to yourself?


Graphics programming is HARD. Expect to spend several times more effort to make a graphical page than a standard HTML page.

Why do you want to do this to yourself?


A graphical UI can make it difficult or impossible for visually impaired users to properly use your site.

Why do you want to do this to yourself?


Graphics will be difficult or impossible for search engines to properly index, potentially harming your SEO.

So, what options do we have to make graphics?


CSS/SVG

OpenGL/WebGL

CSS/SVG


Modern CSS allows powerful animation and transformation capabilities

CSS 3D transformations allow virtual 3D animation

  • Smoothly scales with web page
  • Is widely supported by modern browsers
  • All elements are part of the DOM
  • Element manipulation is easy
  • Modern browsers provide hardware acceleration

CSS/SVG downsides


  • SVG animation can become slow with large numbers of elements
  • CSS transitions are limited in the types of animation they can do
  • CSS 3D transitions are not true 3D graphics
  • E.g.: CSS transitions are probably not good for making a game

Canvas


Canvas is an HTML5 element that allows direct drawing to the screen

  • Allows arbitrary graphics to be drawn
  • Canvas is relatively fast
  • Canvas has excellent browser support
  • Canvas is hardware accelerated in modern browsers
  • Canvas is the method used for almost all HTML5 games
  • Canvas allows for multiple drawing surfaces allowing the generation of sprites

Canvas downsides


  • Canvas is a single DOM element - the internal graphics elements are not part of the DOM
  • Canvas graphics need to be redrawn
  • You need to do all the accounting to manipulate Canvas graphics
  • Any 3D graphics have to be manipulated in Javascript

OpenGL/WebGL


WebGL is a new extension of the Canvas standard that allows full 3D graphics

  • WebGL is an extension of the OpenGL hardware accelerated graphic standard
  • WebGL provides full 3D graphics support including hardware acceleration of the 3D calculations
  • WebGL also provides hardwared accelerated lighting and textures
  • Fully featured modern games can be made in WebGL

WebGL downsides


  • WebGL support is currently not great
  • WebGL sufers from graphics card driver compatibility issues with browsers
  • Getting full WebGL functionality in a browser often requires the user modify advanced browser settings
  • WebGL graphics are also not DOM integrated

So, what to use?

First, identify what sort of application you have.

  • A set of real-time graphs that smoothly displays changing data
  • A 2D retro-style RPG game
  • A 3D music visualization app
  • A graph with several thousand points
  • A presentation software suite

A set of real-time graphs that smoothly displays changing data

  • SVG manipulation via CSS would work best
  • This gives smooth vector graphics
  • All of the elements are part of the DOM and can easily be recolored and resized via CSS

A 2D retro-style RPG game

  • Regular Canvas is recommended
  • Will work in most browers
  • Canvas sprite handling functions make animating small characters easy

A 3D music visualization app

  • WebGL would be recommended here
  • WebGL support of 3D animation allows very high performance graphics

A graph with several thousand points

  • A mixture of SVG elements/CSS transformations and a Canvas element is recommended
  • SVG elements for the axes and labels allow smooth, simple, rescalable UI elements
  • SVG elements will be too slow if thousands are required
  • A Canvas element layered under the SVG allows for fast visualization of the data points

A presentation software suite

  • CSS 3D transformations allow for simple yet impressive transition effects
  • All elements remain part of the DOM
  • The presentation software for this talk RevealJS uses CSS 3D transforms
